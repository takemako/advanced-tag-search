# Advanced Tag Search - 更新履歴

## バージョン 1.8.5 (2026-06-23)

### 変更
- **タグ選択を単一選択に変更**: 検索モーダルでタグを選ぶと1つだけ選択された状態になります
  - 別のタグをクリックすると、以前の選択は自動的に解除されます
  - 選択中のタグをもう一度クリックすると解除されます
  - シンプルで分かりやすい絞り込み体験を優先

## バージョン 1.8.4 (2026-06-23)

### 変更
- **検索モーダルからカテゴリー絞り込みを削除**: 「カテゴリーから絞り込む」セクションを廃止し、タグ検索に一本化しました
  - 表示・カウントが不安定だったカテゴリー絞り込みを取り除き、動作を安定化
  - 管理画面の「カテゴリー絞り込み設定」と、表示順設定のカテゴリー項目も削除
  - クイックリンク（検索窓の下のカテゴリーリンク）はこれまでどおり利用可能

## バージョン 1.8.3 (2026-06-23)

### 変更
- **記事のないタグ・カテゴリーを選択不可に**: 検索モーダルで記事数が0件のタグ・カテゴリーボタンを薄く表示し、クリックできないようにしました
  - 「タグは選べるのに、選んでも記事が表示されない」という状態を回避
  - Kawagoe.fun のような視覚で、選択可能なタグが一目でわかります
  - 対象は「タグカテゴリー内のタグ」と「カテゴリーから絞り込む」の両方

## バージョン 1.8.2 (2026-06-23)

### 変更
- **複数タグの絞り込みを AND から OR に変更**: 複数のタグを選択した場合、すべてのタグを含む記事（AND）ではなく、いずれかのタグを含む記事（OR）をまとめて表示するようにしました
  - これまでは2つ以上のタグを選ぶと、両方を持つ記事が無く0件になることがありました
  - 同種のカテゴリー選択も OR、タグとカテゴリーをまたぐ場合は AND（絞り込み）
  - 件数表示Ajaxも同じ挙動に合わせ、表示件数と検索結果を一致させています

## バージョン 1.8.1 (2026-06-23)

### 不具合修正
- **日本語スラッグの絞り込みで0件になる問題を修正**: 「レビュー」などスラッグがURLエンコードされる日本語タグ・カテゴリーで絞り込むと、検索結果が0件になっていた問題を修正
  - 原因: 単一タグ時に WordPress 標準の `?tag=` 処理へ委ねており、`tag` クエリ変数の再サニタイズで日本語スラッグが壊れていた
  - 対応: タグ／カテゴリーの絞り込みを `tax_query`（スラッグ・AND）に統一し、件数表示（Ajax）と実際の検索結果の挙動を一致させた

## バージョン 1.8.0 (2026-06-23)

### 新機能

```php
public function suppress_tag_errors() {
    // tagパラメータがある場合のみ、エラーハンドラーを設定
    if (isset($_GET['tag']) && !empty($_GET['tag'])) {
        // エラーハンドラーを設定して、ogp.php関連のエラーを抑制
        set_error_handler(function($errno, $errstr, $errfile, $errline) {

            // その他のエラーは通常処理
            return false;

    }
}
```

## バージョン 1.4.0 (2025-11-19)

### 重要な変更
- **v1.2.0の検索機能をベースにエラー対応を追加**: 検索機能を壊さずにエラーを防止
- **`get_queried_object`フィルターを追加**: テーマの`ogp.php`エラーを根本的に解決
- **タグパラメータを修正しない**: 検索機能に影響を与えないエラー対応

### 新機能

**テーマ互換性の向上**

テーマの`ogp.php`が`get_queried_object()`を呼び出した際に、適切なタグオブジェクトを返すようになりました。

**動作:**
1. タグが見つかる場合: 正しいタグオブジェクトを返す
2. タグが見つからない場合: ダミーのタグオブジェクトを返す（`term_id = 0`）

**メリット:**
- テーマが`$tag->term_id`にアクセスしてもエラーが発生しない
- タグパラメータを修正・削除しないため、検索機能に影響なし

### 修正された問題

**問題: テーマの`ogp.php`でWarningが表示される**
```
Warning: Trying to access array offset on false in ogp.php on line 76
Warning: Attempt to read property "term_id" on null in ogp.php on line 76
```

**原因:**
- テーマが`get_queried_object()`を呼び出した際、タグが見つからず`false`が返される
- テーマが`$tag->term_id`にアクセスしてエラーが発生

**解決:**
- `get_queried_object`フィルターで介入
- タグが見つからない場合でも、ダミーのタグオブジェクトを返す
- テーマが安全に`$tag->term_id`にアクセスできる

### 技術的な変更

**追加したフィルター:**
```php
add_filter('get_queried_object', array($this, 'fix_queried_object_for_tags'), 10, 1);
```

**実装:**
```php
public function fix_queried_object_for_tags($queried_object) {
    // 管理画面はスキップ
    if (is_admin()) {
        return $queried_object;
    }
    
    // tagパラメータが存在し、かつqueried_objectがfalseまたはnullの場合
    if (isset($_GET['tag']) && !empty($_GET['tag']) && (!$queried_object || $queried_object === false)) {
        // タグパラメータから最初のタグを取得
        $tags_string = sanitize_text_field($_GET['tag']);
        $tags = array_map('trim', explode(',', $tags_string));
        $tags = array_filter($tags);
        
        if (!empty($tags)) {
            $tag_slug = sanitize_title($tags[0]);
            $tag = get_term_by('slug', $tag_slug, 'post_tag');
            
            // タグが見つかった場合、それを返す
            if ($tag && !is_wp_error($tag)) {
                return $tag;
            }
            
            // タグが見つからない場合、ダミーのタグオブジェクトを返す
            return (object) array(
                'term_id' => 0,
                'name' => '',
                'slug' => '',
                'term_group' => 0,
                'term_taxonomy_id' => 0,
                'taxonomy' => 'post_tag',
                'description' => '',
                'parent' => 0,
                'count' => 0,
            );
        }
    }
    
    return $queried_object;
}
```

## バージョン 1.3.0 (2025-11-19)

### 重要な変更
- **検索機能をv1.1.4の動作に復元**: エラー対応で壊れた検索機能を完全に修正
- **エラー対応のフィルタリングを無効化**: 過剰なフィルタリングが検索を妨げていた
- **シンプルな実装に戻す**: WordPressのデフォルト動作を尊重

### 修正された問題

**問題1: 1つのタグで検索するとトップ画面に戻る**
- **原因**: エラー対応のフィルタリングがタグパラメータを削除していた
- **解決**: フィルタリングを無効化し、WordPressのデフォルト動作に任せる

**問題2: 複数タグで検索すると「該当する記事が見つかりませんでした」になる**
- **原因**: エラー対応のフィルタリングがタグを過剰に修正していた
- **解決**: フィルタリングを無効化し、シンプルなAND検索のみを実装

### 技術的な変更

**無効化したフック:**
```php
// エラー対応のフィルタリング（無効化）
// add_action('init', array($this, 'filter_tag_parameter'), 1);
// add_action('parse_request', array($this, 'filter_tag_parameter_late'), 5);
```

**シンプルな実装:**
```php
public function modify_tag_query($query) {
    // 管理画面やメインクエリ以外はスキップ
    if (is_admin() || !$query->is_main_query()) {
        return;
    }
    
    // tagパラメータが存在するか確認
    if (!isset($_GET['tag']) || empty($_GET['tag'])) {
        return;
    }
    
    $tags_string = sanitize_text_field($_GET['tag']);
    $tags = array_map('trim', explode(',', $tags_string));
    $tags = array_filter($tags);
    
    // タグが1つだけの場合はWordPressに任せる
    if (count($tags) === 1) {
        return;
    }
    
    // 複数タグのAND検索を設定
    $tag_ids = array();
    foreach ($tags as $tag_slug) {
        $tag_slug = sanitize_title($tag_slug);
        $tag = get_term_by('slug', $tag_slug, 'post_tag');
        
        if ($tag && !is_wp_error($tag)) {
            $tag_ids[] = $tag->term_id;
        }
    }
    
    if (!empty($tag_ids)) {
        $query->set('tag__and', $tag_ids);
        $query->set('tag', '');
    }
}
```

**テストケース1: 1つのタグで検索**
- タグを選択: 「川越市駅」
- 結果: 「川越市駅」タグが付いた記事一覧が表示される ✅

**テストケース2: 複数タグで検索**
- タグを選択: 「川越市駅」と「ラーメン」
- 結果: 両方のタグが付いた記事が表示される（AND検索） ✅

### 注意事項

**テーマのogp.phpエラーについて:**

v1.2.1〜1.2.4で対応していたテーマの`ogp.php`エラーは、検索機能を優先するため、本バージョンでは対応していません。

エラーが表示される場合は、以下の対応をお願いします:

1. **WordPressのデバッグモードを無効化**


2. **テーマの`ogp.php`を修正**
   タグ情報にアクセスする前に存在確認を追加してください。
