# Advanced Tag Search - 使用ガイド

## プラグインの概要

Advanced Tag Searchは、美しく使いやすい検索機能をWordPressサイトに追加するプラグインです。訪問者は、モーダルウィンドウでタグを選択して、目的のコンテンツを簡単に見つけることができます。

## 主な機能

### 1. 検索窓
- トップページに配置できる美しい検索窓
- プレースホルダーテキストのカスタマイズ可能
- 検索ボタン付き

### 2. モーダルウィンドウ
- 検索窓をクリックすると開く
- タグをカテゴリー別に整理して表示
- 複数タグの選択が可能
- レスポンシブデザイン

### 3. タグカテゴリー
以下の6つのカテゴリーでタグを分類:
- ジャンルから絞り込む
- エリアから絞り込む
- キーワードから絞り込む
- 時間帯から絞り込む
- 季節から絞り込む
- その他から絞り込む

### 4. クイックリンク
検索窓の下に、よく使うカテゴリーやページへのリンクを表示できます。

### 5. 管理画面
WordPress管理画面から簡単に設定を変更できます。

## インストール手順

### ステップ1: プラグインのインストール

1. `advanced-tag-search.zip` をダウンロード
2. WordPress管理画面にログイン
3. 「プラグイン」→「新規追加」をクリック
4. 「プラグインのアップロード」をクリック
5. ZIPファイルを選択してアップロード
6. 「今すぐインストール」をクリック
7. 「プラグインを有効化」をクリック

### ステップ2: 初期設定

1. WordPress管理画面で「Tag Search」メニューをクリック
2. 基本設定を行う:
   - 検索窓のプレースホルダー
   - モーダルのタイトル

### ステップ3: タグカテゴリーの設定

1. 各カテゴリーのタグをカンマ区切りで入力
2. 例: `食べ歩き, レストラン, カフェ, スイーツ, 居酒屋`

### ステップ4: クイックリンクの設定

1. 「リンクを追加」ボタンをクリック
2. リンクテキストとURLを入力
3. 複数のリンクを追加可能

### ステップ5: 設定を保存

「設定を保存」ボタンをクリックして設定を保存します。

## ショートコードの使い方

### 基本的な使用

ページや投稿に以下のショートコードを追加:

```
[advanced_search]
```

### カスタマイズ例

#### 例1: プレースホルダーを変更

```
[advanced_search placeholder="記事を検索"]
```

#### 例2: クイックリンクを非表示

```
[advanced_search show_quick_links="false"]
```

#### 例3: 両方をカスタマイズ

```
[advanced_search placeholder="記事を検索" show_quick_links="false"]
```

### クイックリンクのみ表示

```
[search_quick_links]
```

## トップページへの配置

### 方法1: ページエディタで配置

1. 「固定ページ」→「トップページ」を編集
2. ショートコードブロックを追加
3. `[advanced_search]` を入力
4. ページを更新

### 方法2: テーマファイルで配置

テーマの `front-page.php` または `home.php` を編集:

```php
<div class="search-section">
    <h2>川越の気になる!を探す</h2>
    <?php echo do_shortcode('[advanced_search]'); ?>
</div>
```

## タグとカテゴリーの準備

### タグの作成

1. 「投稿」→「タグ」をクリック
2. プラグインで設定したタグ名と同じ名前でタグを作成
3. 記事にタグを割り当て

**重要**: プラグインで設定したタグ名とWordPressのタグ名は完全に一致させてください。

例:
- プラグイン設定: `食べ歩き`
- WordPressタグ: `食べ歩き` ✓
- WordPressタグ: `食べ歩き ` ✗（末尾にスペース）

### カテゴリーの作成

1. 「投稿」→「カテゴリー」をクリック
2. 必要なカテゴリーを作成
3. 記事にカテゴリーを割り当て

## 検索機能の動作フロー

### ユーザーの操作フロー

1. **検索窓をクリック**
   - モーダルウィンドウが開く

2. **タグを選択**
   - カテゴリーから目的のタグをクリック
   - 複数のタグを選択可能
   - 選択したタグは色が変わる

3. **絞り込みボタンをクリック**
   - 「選択したタグで絞り込む」ボタンをクリック
   - 選択したタグを含む記事一覧ページへ遷移

4. **検索結果を閲覧**
   - 該当する記事が一覧表示される

### 検索URLの形式

```
https://yoursite.com/?tag=食べ歩き,カフェ
```

複数のタグを選択した場合、カンマ区切りでURLに含まれます。

## カスタマイズ方法

### デザインのカスタマイズ

テーマの `style.css` または追加CSSに以下のコードを追加:

#### 検索ボタンの色を変更

```css
.ats-search-button {
    background: #your-color !important;
}

.ats-search-button:hover {
    background: #your-hover-color !important;
}
```

#### 選択されたタグの色を変更

```css
.ats-tag.selected {
    background: #your-bg-color !important;
    border-color: #your-border-color !important;
    color: #your-text-color !important;
}
```

#### 絞り込みボタンの色を変更

```css
.ats-search-submit {
    background: #your-bg-color !important;
    color: #your-text-color !important;
}

.ats-search-submit:hover {
    background: #your-hover-color !important;
    color: #your-hover-text-color !important;
}
```

#### 全体の色を変更（CSS変数を使用）

```css
:root {
    --ats-primary-color: #A0616A;      /* メインカラー */
    --ats-secondary-color: #E8B4B8;    /* セカンダリカラー */
    --ats-text-color: #333;            /* テキストカラー */
    --ats-bg-color: #FFF;              /* 背景カラー */
}
```

### 検索窓のサイズを変更

```css
.ats-search-container {
    max-width: 1000px; /* デフォルトは800px */
}
```

### モーダルのサイズを変更

```css
.ats-modal {
    max-width: 900px; /* デフォルトは700px */
}
```

## よくある質問

### Q1: タグが表示されません

**A**: 以下を確認してください:
1. 管理画面でタグカテゴリーが正しく設定されているか
2. WordPressにタグが登録されているか
3. タグ名が完全に一致しているか（スペースや大文字小文字に注意）

### Q2: モーダルが開きません

**A**: 以下を確認してください:
1. jQueryが正しく読み込まれているか
2. ブラウザのコンソールでJavaScriptエラーがないか
3. 他のプラグインとの競合がないか

### Q3: 検索結果が表示されません

**A**: 以下を確認してください:
1. 記事にタグが割り当てられているか
2. タグ名が完全に一致しているか
3. 記事が公開状態になっているか

### Q4: デザインが崩れます

**A**: 以下を試してください:
1. ブラウザのキャッシュをクリア
2. テーマのCSSとの競合を確認
3. プラグインのCSSが正しく読み込まれているか確認

### Q5: 複数のタグで検索できますか?

**A**: はい、モーダルで複数のタグを選択できます。選択したすべてのタグを含む記事が検索されます。

### Q6: カテゴリーを追加できますか?

**A**: 現在のバージョンでは、6つの固定カテゴリーのみです。将来のバージョンでカスタムカテゴリーの追加機能を検討しています。

### Q7: 検索結果ページをカスタマイズできますか?

**A**: 検索結果ページは、WordPressのテーマの `tag.php` または `archive.php` テンプレートで制御されます。これらのファイルを編集することでカスタマイズできます。

## トラブルシューティング

### 問題: プラグインが有効化できない

**解決策**:
1. PHPバージョンが7.2以上か確認
2. WordPressバージョンが5.0以上か確認
3. サーバーのエラーログを確認

### 問題: ショートコードが表示されない

**解決策**:
1. プラグインが有効化されているか確認
2. ショートコードのスペルが正しいか確認
3. テーマがショートコードをサポートしているか確認

### 問題: スタイルが適用されない

**解決策**:
1. ブラウザのキャッシュをクリア
2. WordPressのキャッシュプラグインをクリア
3. CSSファイルが正しく読み込まれているか確認（開発者ツールで確認）

## ベストプラクティス

### 1. タグの命名規則

- 短く分かりやすい名前を使用
- 一貫性のある命名規則を採用
- 重複を避ける

### 2. タグカテゴリーの整理

- 各カテゴリーに適切な数のタグを配置（5〜15個程度）
- 使用頻度の高いタグを優先的に配置
- 定期的に見直して最適化

### 3. クイックリンクの活用

- 人気のカテゴリーやページへのリンクを配置
- 5個以内に抑える（多すぎると選択が困難）
- 分かりやすいリンクテキストを使用

### 4. パフォーマンス最適化

- 不要なタグを削除
- 画像の最適化
- キャッシュプラグインの使用

## サポート情報

### 必要な情報

問題が発生した場合、以下の情報を準備してください:

- WordPress バージョン
- PHP バージョン
- 使用しているテーマ
- 有効化している他のプラグイン
- エラーメッセージ（あれば）
- ブラウザとそのバージョン

### デバッグモード

問題を特定するために、WordPressのデバッグモードを有効にできます:

`wp-config.php` に以下を追加:

```php
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
define('WP_DEBUG_DISPLAY', false);
```

エラーログは `wp-content/debug.log` に記録されます。

## アップデート情報

プラグインのアップデートは、WordPress管理画面の「プラグイン」メニューから確認できます。

### アップデート前の注意事項

1. サイトのバックアップを取る
2. テスト環境で動作確認を行う
3. カスタマイズしたCSSをバックアップ
